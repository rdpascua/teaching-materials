exciting and welcoming — like the opening cinematic of a game before the tutorial level.

here's the world you're entering."

## Slide 1: Title Slide
**Day 0 — Before the Quest Begins**

- A 10-Minute Briefer

> Speaker notes: This is a quick orientation. No coding today — just context. Keep it fast and energizing. By the end they should have their tools installed and be hyped for Day 1.

---

## Slide 2: Born in 10 Days
**The Origin Story of JavaScript (1995)**

- Created by **Brendan Eich** at Netscape in just **10 days**
- Originally called "Mocha" → "LiveScript" → "JavaScript"
- Named "JavaScript" to ride Java's hype — **they are not related**

> Speaker notes: Emphasize the 10 days thing — it's wild. That speed is why JS has some quirks, but also why it was agile enough to win the browser war. The Java naming confusion still trips people up today.

---

## Slide 3: The Timeline
**From pop-up scripts to running the world**

```
1995  Created in 10 days
1997  Standardized as ECMAScript
2005  AJAX era — Google Maps, Gmail
2009  Node.js — JS escapes the browser
2015  ES6 — the modern era begins
2026  You're here. Let's go.
```

> Speaker notes: Don't dwell on each year. Just paint the arc: toy language → real apps → everywhere. ES6 in 2015 is the turning point — that's the JavaScript we'll teach.

---

## Slide 4: JS is Everywhere
**One language, every platform**

| Where | What | Examples |
|-------|------|---------|
| Browser | Interactive websites | Chrome, Firefox, Safari |
| Server | APIs & backends | Node.js, Deno, Bun |
| Mobile | Phone apps | React Native |
| Desktop | Desktop apps | VS Code, Discord |

- Only language that runs **natively in every browser**

> Speaker notes: This is the big selling point. Learning JS isn't learning "one tool" — it's a passport to every platform. VS Code itself is built with JavaScript.

---

## Slide 5: What You Need
**Setup checklist (3 things)**

1. **A browser** — Chrome or Firefox (you already have this)
2. **Node.js** — download from nodejs.org (LTS version)
3. **VS Code** — free editor from code.visualstudio.com

- That's it. No frameworks. No complicated setup.

> Speaker notes: Keep this dead simple. Students should not feel intimidated. If they can install two things, they're ready. Walk them through the Node install if needed.

---

## Slide 6: Your First Line of Code
**Open the browser console and type this:**

```js
console.log("Hello, adventurer!");
```

- `F12` or `Cmd+Option+J` (Mac) / `Ctrl+Shift+J` (Windows)
- You just ran JavaScript. That's it.

> Speaker notes: Have everyone do this RIGHT NOW. It takes 10 seconds. The dopamine of seeing output appear is important — it makes JS feel real immediately.

---

## Slide 7: VS Code Extensions
**Install these three before Day 1:**

- **ESLint** — catches mistakes as you type
- **Prettier** — auto-formats your code so it looks clean
- **Live Server** — preview HTML in the browser (needed from Day 15)

> Speaker notes: Show them how to install extensions (Cmd+Shift+X, search, click install). These three will save them hours of frustration.

## Slide 9: JavaScript is Quirky
**And that's okay**

```js
typeof null        // "object"  — a 28-year-old bug
[] + []            // ""        — empty string??
"b" + "a" + +"a"  // "baNaN"   — yes, really
```

- These are fun trivia, not daily problems
- The language is powerful, practical, and runs the modern web

> Speaker notes: Get a laugh here. Show these in the console live. Then reassure them — you won't hit this stuff in real work. It's just the game's lore.
